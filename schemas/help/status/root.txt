Status JSON schema guide

Use this guide when CI, scripts, or external tooling need the machine-readable project-status document.

Contract:

- kind = grafana-util-project-status -> grafana-util-project-status
  schemaVersion = 1
  top-level keys: kind, schemaVersion, toolVersion, scope, overall, domains, topBlockers, nextActions

Quick lookups:
- grafana-util status --help-schema
- grafana-util status staged --help-schema
- grafana-util status live --help-schema

Notes:
- `status` and `overview` reuse the same ProjectStatus wire shape.
- `staged` and `live` emit the same top-level envelope; they differ only in how the fields are populated.
- Branch on kind plus schemaVersion before reading nested fields.
